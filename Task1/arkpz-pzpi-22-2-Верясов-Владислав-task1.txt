1 Розробка VISION & SCOPE


Назва: «Система для підтримки та регуляції прокату електротранспорту»
Назва англійською мовою: «Electric Transport Rental Management System»
Власна назва: «E-Transport».


Розділ 1: Опис проблематики та аналіз аналогів


1.1	Актуальність проблеми:

З розвитком міської мобільності електротранспорт, зокрема електросамокати й велосипеди, став популярним вибором для коротких поїздок. Проте ефективне управління великим парком електротранспорту потребує інноваційного підходу. 
Ключовими проблемами є:
-	Відсутність єдиної платформи для контролю над прокатними засобами (їхнє місцезнаходження, стан, необхідність зарядки або ремонту).
-	Складність у відстеженні стану кожної одиниці та її технічного обслуговування.
- Незручність для користувачів, які не завжди можуть швидко знайти справний та заряджений транспортний засіб поблизу.

1.2	Аналіз аналогів:

Подібні рішення вже застосовуються такими компаніями, як Bird, Lime, та Bolt. Вони пропонують схожий функціонал, однак мають обмеження на адаптацію під індивідуальні потреби та надання зворотного зв'язку щодо кожної одиниці транспорту. На ринку також існують інструменти для моніторингу транспорту, які проте не поєднують функції моніторингу, обслуговування та управління у комплексному рішенні.

1.3 Способи монетизації:

Плата за користування транспортом: Користувачі сплачують за оренду транспортного засобу за хвилини або інші часові інтервали.
Продаж прав на використання інформаційної системи: Доступ до системи можно буде придбати та використовувати власникам парку електротранспорту.
Абонентські пакети: Доступ до системи управління може надаватись через підписки для власників парку електротранспорту.
Інтеграція з іншими послугами: Наприклад, реклама на платформі, партнерства із зарядними станціями.


Розділ 2: Призначення системи


2.1 Призначення системи:

Система створена для моніторингу, управління та обслуговування електротранспортних засобів у прокаті. Вона має забезпечити інтерактивний інтерфейс для користувачів транспорту, зручний доступ до інформації для власників парку електротранспорту, а також інструменти для персоналу, що займається обслуговуванням транспорту.




2.2 Опис бізнес-логіки системи:

Основна логіка системи полягає у тому, що кожна одиниця транспорту відстежується в режимі реального часу, користувачі отримують дані про доступні одиниці поблизу, власники бізнесу можуть керувати запасами, а технічний персонал має доступ до інформації про стан кожної одиниці для обслуговування.


Розділ 3: Функціональні можливості системи


3.1 Функціональність серверної частини та IoT-клієнта:

Серверна частина: Зберігання та обробка даних про транспортні засоби, зокрема їхні локації, стан заряду батареї, технічний стан, історія обслуговування.
IoT-клієнт: Підключення до кожного транспортного засобу для збору даних, які передаються на сервер. Використання датчиків для відстеження стану батареї, місцезнаходження та інших параметрів.

3.2 Функціональність веб-клієнта та мобільного клієнта:

Веб-клієнт: Інтерфейс для власників бізнесу та персоналу. Можливості: управління парком транспорту, перегляд стану кожного засобу, додавання або видалення одиниць, складання звітів.
Мобільний клієнт: Інтерфейс для користувачів транспорту. Можливості: пошук найближчого транспорту, перевірка стану, замовлення, завершення поїздки, перегляд історії використання.



Розділ 4: Сторони, зацікавлені в проєкті, та технології


4.1 Зацікавлені сторони:

Користувачі електротранспорту: Особи, що використовують електротранспорт у повсякденному житті.
Власники парку електротранспорту: Компанії та підприємці, що надають послуги прокату.
Персонал з обслуговування: Співробітники, які займаються зарядкою, ремонтом та управлінням транспортними засобами.

4.2 Технології, які будуть використані у проєкті:

Сервер: фреймворк NestJs та мова програмування TypeScript.
База даних: PostgreSQL для зберігання даних.
IoT-технології: Мікроконтролери для збору даних із транспортних засобів.
Фронтенд: HTML, CSS (Bootstrap), JavaScript (React для веб-клієнта, React Native або Flutter для мобільного).
Хмарні технології: AWS або Koyeb для зберігання даних та обробки інформації в реальному часі.
